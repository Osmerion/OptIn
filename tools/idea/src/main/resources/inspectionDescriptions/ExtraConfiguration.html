Ensure that extra opt-ins and extra opt-in requirements refer to known, valid types.

When configuring global opt-ins or extra opt-in requirements via tooling directly (e.g., via javac) or indirectly (e.g., via Gradle, Maven), all references should refer to existing types fit for their purpose. Global opt-ins must refer to valid requirement markers. Similarly, extra opt-in requirements should refer to annotations that are no requirement markers.
